简单梳理一下JavaScript原型链。

JavaScript原型链

1 JavaScript创建对象的方式

  • 字面量

    let obj1 = {name:'liutao'};
let obj2 = new Object({name:'liutao'});

  • 构造函数

    let M = function(name){
  this.name = name;
}
let obj3 = new M('liutao')

  • create方法

    let obj = {name:'liutao'}
let obj4 = Object.create(obj)

2 相关的关键字

  • 实例对象
  • 原型对象
  • 构造函数
  • Function构造函数
  • Object构造函数
  • Function原型
  • Object原型

3 联系

3.1 设定（约定俗成）

  • 每一个实例对象（除了null）都存在__proto__属性
  • 每一个函数都有一个prototype属性，指向其构造的对象的原型
  • 每一个原型都有一个constructor，指向其关联的构造函数

3.2 基本原则

  • ==实例对象==的==__proto__==属性指向==其原型对象==，实例对象的==构造函数==的==prototype==也指向==该==实例对象的==原型对象==

  • 原型对象也有原型，原型链的==起点==为具体的==实例对象==，终点为==Object==，Object的原型为null，即没有原型对象

  • 一个对象的==原型对象==的==constructor==属性指向==该==对象的==构造函数==

  • 利用原型链可以进行多重继承

3.2 实例的__proto__属性和prototype属性

let obj1 = {name:'liutao'};    //字面量创建一个实例对象
let M = function(name){     //M是构造函数
  this.name = name;
}
let obj2 = new M('liutao');    //构造函数创建一个实例对象
console.log(obj1.__proto__);//{}，即obj1的原型对象
console.log(obj2.__proto__);//M{}，即obj2的原型对象
console.log(obj2.__proto__===M.prototype)//通过3.1可知M.prototype指向其构造的对象即obj2的原型
//这里可以得出，字面量实例对象的__proto__属性和构造函数实例对象的__proto__属性指向各自的原型对象

3.3 原型对象的__proto__属性和prototype属性

console.log(obj1.__proto__.__proto__);//null，由于obj1是由Object构造函数创建的，所以obj1.__proto__指向Object的prototype，而obj1.__proto__.__proto__指向Object的原型，即null
console.log(obj2.__proto__.__proto__);//{}，由于obj2是由M构造函数创建的，所以obj2.__proto__指向M.prototype，而M是由Object构造函数创建的，所以obj2.__proto__.__proto__指向M.prototype.__proto__，即Object.__proto__，即obj1.__proto__
//验证
console.log(obj2.__proto__.__proto__ === obj1.__proto__);//true
console.log(obj2.__proto__.__proto__.__proto__);//false

总结

原型链的图其实画的不太好，但是只要理解了3.1和3.2的内容，就很容易理解原型链了。
