学习WebPack工具的一些前端基础，这是web前端系列的第一篇文章。

CommonJS

//导出
//test.js
//方式1
module.exports = {
  name:'module1',
  add: function(a,b){
    return a + b
  }
}
//方式2
exports.name = 'module1';
exports.add = function(a,b){
  return a + b
}

//导入
//index.js
//方式1
const test = require('./test.js')
const add = test.add
//方式2，支持表达式动态导入
const modules = ['test.js','test2.js']
modules.forEach(name=>{
  require('./'+name)
})

CommonJS注意的地方

  • 在实际使用中，为了提高可读性，建议将module.exports和exports语法语句放在模块最后
  • exports指向module.exports，因此不要直接给exports对象赋值，否则会导致module.exports导出失效
  • require语句在一个js文件中只执行一次，重复使用不会再执行导入模块中的语句
  • 每个模块都有自己的作用域
  • CommonJs导入模块是对其值的拷贝，可以进行修改，不影响原模块

ES6

//导出
//test.js
//方式1，默认导出，只有一个export
export dufault {
  name:'test',
  add:function(a,b){
    return a + b
  }
}
//方式2
export const name = 'test'
export add=function(a,b){
  return a+b
}
//方式3，默认导出
const name = 'test'
add = function(a,b){
  return a+b
}
export {test,add}
//方式3，命名导出
const name = 'test'
add = function(a,b){
  return a+b
}
export {test as test1,add as add1}

//导入
//index.js
//方式1，按照原名导入
import {name,add} from './test.js'
//方式2，命名导入
import {name as name1, add as add1} from './test.js'
//方式3 整体导入
import * from './test.js'
//方式4，对默认导出的导入
import myModule from './test.js' //相当于import default as myModule from './test.js'

ES6注意的地方

  • 每个模块有自身的作用域
  • 所有导入的变量都是原模块的动态映射，是只读的，不可以对其进行修改
  • 对于默认导出来说，导入可以接自定义变量名
  • 导出、导出必须放在顶层作用域

CommonJS和ES6的区别

  • CommonJS对模块依赖的解决是动态的，模块依赖关系的建立发生在代码运行阶段。ES6对模块依赖解决的办法是静态的，模块依赖关系的建立发生在代码编译阶段
  • CommonJS支持导入的路径是表达式，ES6的导入是声明式的，故不支持
  • ES6的好处
    • 死代码检测和排除
    • 模块变量类型检查
    • 编译器优化
    • 可以利用ES6的特性解决循环依赖问题
