1 No.8 字符串转整数

请你来实现一个atio函数，使其能将字符串转换成整数。要求：

  • 首先丢弃字符串开头的空格
  • 非空格的第一个字符串是正负号时，则将该符号与后面尽可能多连续的数字字符组合起来，形成一个有符号整数
  • 非空格的第一个字符是数字时，直接将其与之后连续的数字字符组合起来，形成一个有符号整数
  • 该字符串有效整数部分之后也可能存在多余的字符，应该被忽略
  • 当不满足以上规则而无法进行转换时，返回0
  • 返回整数的大小范围为32位有符号整数的范围

1.1 直接上手

最简单的方法是直接开始编码，但是显然，会遇到很多麻烦，并且写出的代码具有非常多的if-else结构，十分臃肿，我在VS Code上测试了十余次，前后经过了一两个小时，才写出完整AC的代码，可以说是十分low了。

class Solution:
    def myAtoi(self, str: str) -> int:
        res = " "
        flag = True
        strList = list(str.strip())
        for i in range(len(strList)):
            if strList[i] != " ":
                for j in strList[i:]:
                    if j == " ":
                        break
                    if (j == "+" or j == "-") and flag:
                        res += j
                        flag = False
                        continue
                    elif j.isdigit():
                        res += j
                        flag = False
                    else:
                        break
                break
        try:
            res = int(res)
        except:
            return 0
        return max(-2**31, min(res, 2**31-1))

其中flag用于标志是否已经添加了正负号，因为如果对符号进行统一的直接拼接处理，则会如果有多个正负号出现时程序出错的问题。（这其实是测试了多次，从测试用例观察出来的）

整个思路为：

  • 首先将字符串转换成列表，包括空格在内
  • 进入第一个for循环，字符非空格时，进入第二个循环，循环从第一个非空字符开始，即strList[i:]
  • 非空字符后面再遇到空格时，结束循环
  • 如果遇到正负号，则判断flag，仅在第一次出现时添加符号
  • 如果遇到数字字符，则直接和res进行拼接
  • 遇到其他字符，则结束循环
  • 将res转换成整数，为了避免可能出现的问题如res为空等情况，使用try-catch语法将其包裹，如果出错直接返回0
  • 最后判断res是否在32位有符号整数的范围内，根据条件返回整数

算法的时间复杂度为O(n)，空间复杂度为O(1)

可以看到整个流程十分复杂，并且绝不是一次想好就能编写出这样的代码，因此在面对这样的题目时，我们需要用另一种思路去更清晰的分析问题。

1.2 有限状态机

通过简单分析，可以知道整个转换的流程主要有4种状态：

  • 初始状态，直到首个不为空格的字符之前，都处于这个状态
  • 符号转换状态，在遇到符号时，处于这个状态
  • 数字转换状态，在遇到数字字符时，出于这个状态
  • 结束状态，即程序退出

各状态之间的转换流程和其对应的表格如上图所示。根据状态转移图和表格，可以很方便的写出程序：

INT_MAX = 2 ** 31 - 1
INT_MIN = -2 ** 31
class Auto:
    def __init__(self):
        self.status = 'start'
        self.sign = 1
        self.ans = 0
        self.status_matrix = {
            'start': ['start', 'change_number', 'signed', 'end'],
            'signed': ['end', 'change_number', 'end', 'end'],
            'change_number': ['end', 'change_number', 'end', 'end'],
            'end': ['end', 'end', 'end', 'end']
        }
    # 获取当前的状态
    def get_status(self, c: str) -> int:
        if c.isspace():
            return 0
        elif c.isdigit():
            return 1
        elif c == '+' or c == "-":
            return 2
        else:
            return 3
    # 处理当前字符
    def get(self, c: str):
        self.status = self.status_matrix[self.status][self.get_status(c)]
        if self.status == 'change_number':
            self.ans = self.ans*10 + int(c)
            self.ans = min(self.ans, INT_MAX) if self.sign == 1 else min(
                self.ans, -INT_MIN)
        elif self.status == "signed":
            self.sign = 1 if c == '+' else -1

class Solution:
    def myAtoi(self, str: str) -> int:
        auto = Auto()
        for c in str:
            auto.get(c)
        return auto.sign * auto.ans

根据状态转移表创建status_martix字典，以每种状态为键，值依次为下一个字符出现时的转移状态，通过get_status方法处理并返回下一个状态，get处理符号和数字两种情况。

这样一来，整个程序的状态十分清晰，也无需再依靠苦苦的调试测试用例来写一堆臃肿的if-else代码，算法的时间复杂度为O(n)，空间复杂度为O(1)虽然最后通过击败的人数不如前一中方法，但是这样的方式思考问题是值得借鉴的。
