1 No.8 字符串转整数

请你来实现一个atio函数，使其能将字符串转换成整数。要求：

  • 首先丢弃字符串开头的空格
  • 非空格的第一个字符串是正负号时，则将该符号与后面尽可能多连续的数字字符组合起来，形成一个有符号整数
  • 非空格的第一个字符是数字时，直接将其与之后连续的数字字符组合起来，形成一个有符号整数
  • 该字符串有效整数部分之后也可能存在多余的字符，应该被忽略
  • 当不满足以上规则而无法进行转换时，返回0
  • 返回整数的大小范围为32位有符号整数的范围

1.1 直接上手

最简单的方法是直接开始编码，但是显然，会遇到很多麻烦，并且写出的代码具有非常多的if-else结构，十分臃肿，我在VS Code上测试了十余次，前后经过了一两个小时，才写出完整AC的代码，可以说是十分low了。

class Solution:
    def myAtoi(self, str: str) -> int:
        res = " "
        flag = True
        strList = list(str.strip())
        for i in range(len(strList)):
            if strList[i] != " ":
                for j in strList[i:]:
                    if j == " ":
                        break
                    if (j == "+" or j == "-") and flag:
                        res += j
                        flag = False
                        continue
                    elif j.isdigit():
                        res += j
                        flag = False
                    else:
                        break
                break
        try:
            res = int(res)
        except:
            return 0
        return max(-2**31, min(res, 2**31-1))

其中flag用于标志是否已经添加了正负号，因为如果对符号进行统一的直接拼接处理，则会如果有多个正负号出现时程序出错的问题。（这其实是测试了多次，从测试用例观察出来的）

整个思路为：

  • 首先将字符串转换成列表，包括空格在内
  • 进入第一个for循环，字符非空格时，进入第二个循环，循环从第一个非空字符开始，即strList[i:]
  • 非空字符后面再遇到空格时，结束循环
  • 如果遇到正负号，则判断flag，仅在第一次出现时添加符号
  • 如果遇到数字字符，则直接和res进行拼接
  • 遇到其他字符，则结束循环
  • 将res转换成整数，为了避免可能出现的问题如res为空等情况，使用try-catch语法将其包裹，如果出错直接返回0
  • 最后判断res是否在32位有符号整数的范围内，根据条件返回整数

算法的时间复杂度为O(n)，空间复杂度为O(1)

可以看到整个流程十分复杂，并且绝不是一次想好就能编写出这样的代码，因此在面对这样的题目时，我们需要用另一种思路去更清晰的分析问题。

1.2 有限状态机

通过简单分析，可以知道整个转换的流程主要有4种状态：

  • 初始状态，直到首个不为空格的字符之前，都处于这个状态
  • 符号转换状态，在遇到符号时，处于这个状态
  • 数字转换状态，在遇到数字字符时，出于这个状态
  • 结束状态，即程序退出

各状态之间的转换流程和其对应的表格如上图所示。根据状态转移图和表格，可以很方便的写出程序：

INT_MAX = 2 ** 31 - 1
INT_MIN = -2 ** 31
class Auto:
    def __init__(self):
        self.status = 'start'
        self.sign = 1
        self.ans = 0
        self.status_matrix = {
            'start': ['start', 'change_number', 'signed', 'end'],
            'signed': ['end', 'change_number', 'end', 'end'],
            'change_number': ['end', 'change_number', 'end', 'end'],
            'end': ['end', 'end', 'end', 'end']
        }
    # 获取当前的状态
    def get_status(self, c: str) -> int:
        if c.isspace():
            return 0
        elif c.isdigit():
            return 1
        elif c == '+' or c == "-":
            return 2
        else:
            return 3
    # 处理当前字符
    def get(self, c: str):
        self.status = self.status_matrix[self.status][self.get_status(c)]
        if self.status == 'change_number':
            self.ans = self.ans*10 + int(c)
            self.ans = min(self.ans, INT_MAX) if self.sign == 1 else min(
                self.ans, -INT_MIN)
        elif self.status == "signed":
            self.sign = 1 if c == '+' else -1

class Solution:
    def myAtoi(self, str: str) -> int:
        auto = Auto()
        for c in str:
            auto.get(c)
        return auto.sign * auto.ans

根据状态转移表创建status_martix字典，以每种状态为键，值依次为下一个字符出现时的转移状态，通过get_status方法处理并返回下一个状态，get处理符号和数字两种情况。

这样一来，整个程序的状态十分清晰，也无需再依靠苦苦的调试测试用例来写一堆臃肿的if-else代码，算法的时间复杂度为O(n)，空间复杂度为O(1)虽然最后通过击败的人数不如前一种方法，但是这样的方式思考问题是值得借鉴的。

3 删除链表的倒数第n个结点

给定一个链表，删除链表的倒数第n个结点，并返回链表结点的头结点。

链表的缺点就是查找元素不方便，要找到任何一个结点都必须遍历其之前的全部结点，所以一般的思路是：

  • 遍历链表，记录链表的大小
      count=0
  while head:
      count+=1
      head = head.next
  • 利用循环，当遍历到倒数第n个节点的前一个结点时，删除节点
      start = head
  for i in range(0,count):
      if count-i == n-1:
          start.next = start.next.next
      start = start.next
    但是其实可以利用多个指针来进行，这样就无需事先遍历统计节点的个数了。
  • 设定一个快指针fast，一个慢指针slow，指向头结点head，假设总共有N个结点，则倒数第n个结点的索引为N-n-1
  • 先将快指针fast向后移动n次， 指向结点索引为n-1
  • 然后将快指针fast移动到末尾，同时移动慢指针slow，当fast指向末尾结点时，慢指针指向结点索引为N-n，即倒数第n个结点的前一个结点
  • 此时直接删除慢指针slow指向的结点的后一个结点即可
    图示如下：
    class Solution:
  def removeNthFromEnd(self, head: ListNode, n: int) -> ListNode:
      fast = slow = head
      for _ in range(n):
          fast = fast.next
      if not fast:
          return head.next
      while fast.next:
          fast = fast.next
          slow = slow.next
      slow.next = slow.next.next
      return head

    4 括号生成

    数字n代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且有效的括号组合。

4.1 回溯法（深度优先遍历）

首先想到的就是回溯法（深度优先遍历），由于其可以利用函数递归的方式，直接利用系统栈，而广度优先遍历往往需要我们自定义数据结构。
分析如下：

  • 当左括号的数量小于n时，可以生成子树
  • 产生左子树时，仅判断左括号数量是否小于n
  • 当产生右子树时，判断右括号数量是否小于左括号数量
  • 整个树的节点数量等于2n时结束

以n=2的情况为例，其树的生成情况如图所示：

ans=[]
def generateParenthesis(self, n: int) -> List[str]:
    def generate(Tree,L,R):
        if len(Tree)==2*n:
            ans.append(''.join(Tree))
            # 满足条件的组合进入后，返回到上一层函数
            return
        if L < n:
            Tree.append('(')
            generate(Tree,L+1,R)
            # 得到调用函数的返回时，出栈，回到父函数
            Tree.pop()
        if L > R:
            Tree.appned(')')
            generate(Tree,L,R+1)
            # 得到调用函数的返回时，出栈，回到父函数
            Tree.pop()
    generate([],0,0)
    return ans

4.2 动态规划

首先分析状态及其转移方程：

  • 状态dp[i]代表括号对数为i时的所有正确的组合数
  • 状态dp[i]依赖于dp[j]和dp[k]，其中j和k满足j+k=i-1，事实上，dp[i]仅仅比dp[j]和dp[k]的组合多了一对括号( ),其位置关系为：
    • 从dp[0]开始，dp[0]=[""]
    • dp[j]在( )内部，dp[k]在( )外部
    • 即dp[i] = "(" + dp[j] + ")" + dp[k]
      class Solution:
def generateParenthesis(self, n: int) -> List[str]:
  if n == 0:
      return []
  dp = [None for _ in range(n + 1)]
  dp[0] = [""]
  for i in range(1, n + 1):
      cur = []
      for j in range(i):
          left = dp[j]
          right = dp[i - j - 1]
          for s1 in left:
              for s2 in right:
                  cur.append("(" + s1 + ")" + s2)
      dp[i] = cur
  return dp[n]
